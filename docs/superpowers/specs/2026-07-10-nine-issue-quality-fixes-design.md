# 健身小程序九项体验与数据问题修复设计

日期：2026-07-10

## 目标与范围

本次修改解决底部导航、个人中心、头像上传、体重记录、训练计划编辑、数据图表、动作表现、饮食日期和账号设置共九项问题。改动沿用现有 Vue 3、uni-app、FastAPI、SQLAlchemy 和本地上传目录，不引入新的 UI 或存储依赖。

成功标准：页面选中状态稳定；用户头像可从设备选择并长期保存；体重记录可折叠且编辑、删除语义准确；训练计划输入内容完整可见；统计图和动作指标使用真实训练数据；饮食日期易于识别；账号页不再混入体重业务。

## 设计决策

采用“共享逻辑修复 + 页面局部重设计”。不重构整个应用，也不只覆盖表面样式。导航、统计和上传在其数据源处修复；页面结构只调整与本次问题直接相关的部分。

## 1. 底部导航

当前页面路由是底栏选中状态的唯一可信来源。页面展示时同步当前索引，点击时只发起 `switchTab`，不先写入可能残留到下一页的局部状态。导航进行期间忽略重复点击，成功或失败后解除锁定。

药丸位移使用浏览器和微信小程序均可识别的百分比变换值，例如索引 2 对应 `translateX(200%)`，不生成包含乘法表达式的无效 CSS。页面首次显示时不播放跨越多个标签的过渡动画；仅同一底栏实例内、状态已同步后的用户点击允许正常过渡。

## 2. “我的”页面图标

功能入口统一使用现有 `LineIcon` 和底栏的 SVG 图标源。去除彩色玻璃方块和多余渐变，保持简洁线性描边；颜色使用主色或中性色表达层级，危险操作仍使用红色。

## 3. 基础资料头像上传

基础资料页用圆形头像预览和“选择图片/更换头像”入口替代头像 URL 输入框。流程为：

1. 用户通过 `uni.chooseImage` 选择一张图片。
2. 前端调用现有图片上传接口，`usage_type` 使用新增的 `avatar`，`temporary=false`。
3. 后端沿用文件类型、大小和内容校验，将文件保存到本地上传目录，并返回相对静态地址。
4. 上传成功后将返回地址写入资料表单；用户保存资料时，用户表只保存该地址。

选择、上传或保存失败时保留原头像并显示错误提示。取消选择不改变资料。此次不实现裁剪、云存储和历史头像自动清理。

## 4. 体重记录

“记录体重”继续从“我的”页面独立进入，不属于“账号与数据”。最近记录区域默认折叠，标题行显示记录数量和展开/收起按钮。展开后按时间倒序显示记录。

每条记录提供两个独立按钮：

- 编辑：将记录的体重、日期、时间和备注载入输入区，并把主按钮切换为保存修改；支持取消编辑。
- 删除：显示包含日期和体重的二次确认框，确认后仅删除该记录。

新增、编辑或删除成功后重新加载最新体重、趋势图和记录列表。

## 5. 训练计划编辑输入框

沿用现有双列表单布局，但为原生 `input` 设置明确的可用高度、行高、左右内边距、盒模型和最小宽度。标签与输入值不得重叠；较长动作名优先保证输入内容可横向完整编辑，页面不通过缩小字体掩盖问题。

同时检查微信小程序端的 scoped 样式和原生输入控件表现，确保动作名、部位、组数、次数、重量、休息六类字段一致。

## 6. 数据页训练与体重图表

前端先把接口序列转换为稳定的日期、训练次数、训练容量和体重数据，再交给图表组件。图表初始化必须发生在容器完成布局后，并在页面重新显示或时间范围变化后更新。

展示规则：

- 存在训练数据：绘制训练次数与容量，摘要显示区间合计。
- 存在体重数据：只连接有效体重点，不把缺失日期转换成 0 kg。
- 没有业务数据：显示带说明和入口的空状态，不保留空白图表画布。
- 接口失败：显示加载失败和重试，不伪装成 0。

## 7. 动作表现统计

统计范围只包含已完成训练和已完成训练组。单组的有效值按以下优先级确定：

- 重量：有效的实际重量；旧记录缺失时回退到有效的目标重量。
- 次数：有效的实际次数；旧记录缺失时回退到有效的目标次数。
- 组容量：有效重量 × 有效次数。

动作最高重量为所有有效组的最大重量；总容量为所有有效组容量之和。无负数或非数值参与计算。确实没有重量数据的自重动作显示“自重/暂无重量”，不误报为具有意义的 `0 kg`。

## 8. 饮食日期区域

顶部改为两层日期结构：第一层突出当前选择日期的完整月日和语义，例如“7月10日 · 今天”；第二层提供相邻七天的星期与日号。非本月日期同时显示月/日，避免跨月误解。今天和选中态分别表达，用户可以识别“今天”以及“当前正在查看哪天”。

保留左右滑动或点击日期切换能力，不增加月历弹窗。

## 9. 账号与数据页面

普通模式彻底移除体重显示、录入、趋势及相关自动滚动逻辑。页面分组为：

- 账号安全：手机号授权或账号信息。
- 数据与存储：数据导出、清除缓存。
- 隐私与协议：用户协议、隐私政策。
- 危险操作：删除个人数据、注销账号。

尚未实现的功能保持明确禁用状态并标注“开发中”，不能响应点击。危险操作保留二次确认。体重专用模式仍可暂时复用同一路由参数，但界面和逻辑与普通账号页严格分支；若后续单独拆页，不属于本次范围。

## 数据与接口变化

- 图片上传接口允许 `usage_type=avatar`，头像必须以永久文件上传。
- 用户资料更新接口沿用 `avatar_url` 字段，无需数据库迁移。
- 训练统计接口响应结构保持不变，修复内部聚合数据来源和回退规则。
- 不新增第三方依赖。

## 错误处理

所有异步操作必须解除 loading 或导航锁。上传、统计和体重操作失败时保留当前可用数据并给出明确提示。统计接口不能把异常吞掉后返回全零；前端区分“无数据”和“加载失败”。

## 测试与验收

先添加失败测试，再实现修复：

- 导航索引到位移值的计算、路由到索引的同步和重复点击锁。
- 头像用途允许、永久上传及非法图片拒绝。
- 动作实际数据聚合、旧数据回退和自重动作场景。
- 训练与体重序列在有数据、缺失日期和空数据时的转换。
- 体重编辑载入与删除操作保持独立。

完成后运行后端 pytest、前端 Vitest、TypeScript/构建检查，并生成微信小程序构建产物。对九个页面按用户提供的复现路径进行人工回归，特别检查跨标签点击、输入框真机样式和图表非空显示。

## 非目标

本次不接入对象存储、不增加图片裁剪、不重做全部视觉系统、不迁移历史上传文件、不实现手机号授权或数据导出的后端能力，也不拆分新的体重页面路由。
